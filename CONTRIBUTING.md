# 🤝 CONTRIBUTING — Data Analyst: 0 to Hero in 16 Weeks

> Este repositorio es un proyecto de aprendizaje público. Si estás siguiendo el programa, encontraste un error, o quieres contribuir con mejoras, esta guía es para ti.

---

## 📌 ¿Quién puede contribuir?

Este proyecto acepta contribuciones de:

- **Estudiantes** que están siguiendo el programa y encontraron errores o mejoras
- **Data Analysts** que quieren agregar recursos o casos de uso adicionales
- **Docentes o mentores** que desean sugerir ejercicios o correcciones técnicas
- **Cualquier persona** interesada en democratizar el aprendizaje de Data Analytics

---

## 🐛 Reportar un error

Si encuentras un error en algún archivo (typo, query incorrecta, fórmula con bug, link roto), por favor:

1. Ve a la pestaña **Issues** del repositorio
2. Haz clic en **New Issue**
3. Usa el siguiente formato:

```
Título: [SEMANA] Tipo de error — descripción breve
Ejemplo: [WEEK02] Query incorrecta en ejercicio 3 de PRACTICA_Week02

Descripción:
- Archivo afectado: PRACTICA_Week02.md
- Línea aproximada: 145
- Error encontrado: [describe el error]
- Comportamiento esperado: [describe qué debería hacer]
- Sugerencia de corrección (opcional): [tu propuesta]
```

## 💡 Proponer mejoras

¿Tienes una idea para mejorar el programa? Ejercicios adicionales, casos de uso, datasets más realistas, recursos nuevos. Abre un Issue con el tag `enhancement`:

```
Título: [MEJORA] Descripción corta de la propuesta
Ejemplo: [MEJORA] Agregar ejercicio de análisis de devoluciones en Week02 SQL

Descripción:
- Semana relacionada: Week 02
- Tipo de mejora: Nuevo ejercicio práctico
- Justificación: El análisis de devoluciones es muy común en retail analytics y no está cubierto
- Contenido propuesto: [descripción o borrador]
```

## 🔧 Hacer un Pull Request

Si quieres contribuir directamente con código o contenido:

```bash
# 1. Haz fork del repositorio
# 2. Clona tu fork
git clone https://github.com/TU-USUARIO/Data-Analyst-0-to-Hero-16-Weeks.git

# 3. Crea una rama descriptiva
git checkout -b fix/week02-query-error
# o
git checkout -b feature/week03-ejercicio-adicional

# 4. Haz tus cambios

# 5. Commit con mensaje claro
git commit -m "fix: corrige query de cohortes en PRACTICA_Week02 ejercicio 3"

# 6. Push a tu fork
git push origin fix/week02-query-error

# 7. Abre un Pull Request desde GitHub
```

### Convención de commits

```
fix:      Corrección de error
feat:     Contenido nuevo
docs:     Mejora de documentación
refactor: Reorganización sin cambio de contenido
style:    Formato, espaciado, typos
```

## ✅ Estándares de calidad

Antes de abrir un PR, verifica:

- [ ] El contenido está en español (idioma principal del programa)
- [ ] Los queries SQL están testeados en PostgreSQL
- [ ] Los ejemplos de Python están testeados con Python 3.10+
- [ ] No se incluyen datos personales reales
- [ ] Los archivos Markdown tienen formato limpio y consistente
- [ ] Se mantiene el enfoque en **Retail Analytics**

---

## 📜 Código de Conducta

Este es un espacio de aprendizaje. Se espera:

- **Respeto** — Crítica constructiva, no personal
- **Claridad** — Issues y PRs bien documentados
- **Enfoque** — Contribuciones relevantes al tema del repositorio
- **Honestidad** — Si algo no funciona, dilo con ejemplos concretos
